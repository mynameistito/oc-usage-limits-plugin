# Security Policy

## Reporting

Please report security issues privately through GitHub vulnerability reporting, or email `mynameistito@users.noreply.github.com`.

Include a short description, reproduction steps, affected versions, and impact. Do not open a public issue for vulnerabilities.

We aim to respond within 72 hours.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.x     | Yes       |
| < 1.0   | No        |

## Scope

This project may read local credential files and config, so do not include real credentials, API keys, tokens, or response bodies in reports.

## Acknowledgment

We appreciate responsible disclosure and will handle reports discreetly.
